Verbatim PP-Macros Set

“Verbatim” is a set of PP-macros for handling various types of verbatim text:

  • Preformatted blocks (<pre>)
  • Code blocks with custom classes for non syntax-higlighted coloring (<pre><code class="someclass">)

It looks like I could without the !Code and !CodeSmall macros, and use pandoc’s support for {.class} syntax in fenced blocks (as well as inline code).

But I must check the CSS stylesheets, because pandoc will only put the class on the <pre> tag, and nothing on the <code> tag! If I can fix the CSS to cover nicely both syntax highlighting and preformatted blocks, it would be better to do without these macros.

See: Pandoc Markdown Alternative for examples and test.

Preformatted Blocks

Thre preformatted blocks macros were devised because markdown doesn’t have a syntax for wrapping text so that in the final HTML docs they will be wrapped in <pre> tags (without <code> tags inside).

Markdown Lacks a Syntax for Preformatted Blocks

Markdown has no syntax to define verbatim text that will output as preformatted block only (<pre>) — ie: it only allows for <pre><code> blocks.

There is the four-spaces indentantion syntax for “indented code blocks”, which will wrap the text in <pre><code> tags:

MARKDOWN SOURCE:

A block of text indented four spaces (or one tab) is treated as verbatim text:

    Some verbatim text.
      Whitespace preserved!

HTML OUTPUT:

<p>A block of text indented four spaces (or one tab) is treated as verbatim text:</p>


<pre><code>Some verbatim text.


  Whitespace preserved!</code></pre>

VISUAL RESULT:

A block of text indented four spaces (or one tab) is treated as verbatim text:

Some verbatim text.
  Whitespace preserved!

… which is just like using the “fenced code blocks” syntax without a language specification:

MARKDOWN SOURCE:

```
Some verbatim text.
  Whitespace preserved!
```

HTML OUTPUT:

<pre><code>Some verbatim text.


  Whitespace preserved!</code></pre>

VISUAL RESULT:

Some verbatim text.
  Whitespace preserved!

The !Pre Macro

The !Pre macro will wrap the text block within <pre> tags.

MARKDOWN SOURCE:

!Pre
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
Some verbatim text.
  Whitespace preserved!
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

HTML OUTPUT:

<pre>Some verbatim text.


  Whitespace preserved!


</pre>

VISUAL RESULT:

Some verbatim text.
  Whitespace preserved!

Code Blocks Macros

Pandoc Markdown Alternative

  • http://pandoc.org/MANUAL.html#extension-fenced_code_blocks

MARKDOWN SOURCE:

```{.someclass}
// Some pseudo-code
if x > 10 Then
    do something
```

HTML OUTPUT:

<pre class="someclass"><code>// Some pseudo-code


if x &gt; 10 Then


    do something</code></pre>

VISUAL RESULT:

// Some pseudo-code
if x > 10 Then
    do something

Using the small class:

MARKDOWN SOURCE:

```{.small}
// Some pseudo-code
if x > 10 Then
    do something
```

HTML OUTPUT:

<pre class="small"><code>// Some pseudo-code


if x &gt; 10 Then


    do something</code></pre>

VISUAL RESULT:

// Some pseudo-code
if x > 10 Then
    do something
