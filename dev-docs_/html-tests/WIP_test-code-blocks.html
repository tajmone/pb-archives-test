Preformatted Blocks

In order to achieve a simple <pre> block (not followed by the <code> code) you have to resort to raw HTML tags inside markdown — ie: there is no way to achived it in pure markdown.

MARKDOWN SOURCE: 

<pre>I'm a preformatted block, no code inside</pre>

HTML OUTPUT: 

<pre>I'm a preformatted block, no code inside</pre>

VISUAL RESULT: 

I'm a preformatted block, no code inside

Non-Highlighted Code Blocks

To define a code block without any highlighting, just enclosed it in tildas or backticks (fenced code), or use indentation (4 spaces or one Tab).

Fenced (backticks)

MARKDOWN SOURCE: 

```
For i=1 To 10
  Debug("Counting " + Str(i))
Next
```

HTML OUTPUT: 

<pre><code>For i=1 To 10


  Debug(&quot;Counting &quot; + Str(i))


Next</code></pre>

VISUAL RESULT: 

For i=1 To 10
  Debug("Counting " + Str(i))
Next

Indented (4 spaces)

NOTE — The raw html comment tag <!----> was added to create a context in the snippet, otherwise pandoc would convert it as a paragraph.

MARKDOWN SOURCE: 

<!---->
    For i=1 To 10
      Debug("Counting " + Str(i))
    Next

HTML OUTPUT: 

<!---->


<pre><code>For i=1 To 10


  Debug(&quot;Counting &quot; + Str(i))


Next</code></pre>

VISUAL RESULT: 

For i=1 To 10
  Debug("Counting " + Str(i))
Next

Line blocks

Extension: line_blocks

A line block is a sequence of lines beginning with a vertical bar (“|”) followed by a space. The division into lines will be preserved in the output, as will any leading spaces; otherwise, the lines will be formatted as Markdown. This is useful for verse and addresses:

| The limerick packs laughs anatomical
| In space that is quite economical.
|    But the good ones I've seen
|    So seldom are clean
| And the clean ones so seldom are comical

… which produces (horizontal rulers added):

The limerick packs laughs anatomical In space that is quite economical.    But the good ones I’ve seen    So seldom are clean And the clean ones so seldom are comical

It basicall wraps the contents in a <div style="white-space: pre-line;">.

The lines can be hard-wrapped if needed, but the continuation line must begin with a space.

| The Right Honorable Most Venerable and Righteous Samuel L.
  Constable, Jr.
| 200 Main St.
| Berkeley, CA 94718

… which produces (horizontal rulers added):

The Right Honorable Most Venerable and Righteous Samuel L. Constable, Jr. 200 Main St. Berkeley, CA 94718

Long lines will wrap around:

| A very lengthy and overly-verbose line of text that will surely exceed its container's width by an abundant measure and force wrapping!

… which produces (horizontal rulers added):

A very lengthy and overly-verbose line of text that will surely exceed its container’s width by an abundant measure and force wrapping!
